# خطة إكمال الربط والتسويات

## الحالة الحالية لـ Vogatchi Travel
- 178 حجز، كلها مربوطة بعملاء ✅
- 169 فاتورة "مدفوعة" + 169 سداد مورد "مدفوع" + 353 قيد محاسبي ✅
- **متبقي:** 5 حجوزات بدون مورد، و9 حجوزات بدون فاتورة (سعر بيع = 0)

## الخطوات

### 1. سد فجوات الموردين (5 حجوزات)
- لكل حجز بدون مورد: إنشاء/استخدام مورد افتراضي `Unknown Supplier` لنفس المؤسسة وربطه بالحجز
- تحديث `supplier_id` و `supplier_name` على الحجز
- بدون ذلك لا يمكن إصدار سداد مورد له

### 2. تسوية الحجوزات بدون أسعار (9 حجوزات)
- استعراض الحجوزات اللي `selling_price=0` أو `cost_price=0`
- وضعها في صفحة "جودة البيانات" كتنبيه للمراجعة اليدوية (لن أخمن مبالغ)
- بعد إدخال السعر، التريجر الحالي يُنشئ الفاتورة والسداد تلقائياً عبر زر "تسوية"

### 3. ربط مدفوعات الفواتير بحساب بنكي
- المدفوعات حالياً مسجلة على حساب "النقدية 1000"
- إضافة خيار اختياري لتحديد حساب بنكي افتراضي للمؤسسة، وتحديث القيود اللي اتسجلت لو المستخدم وافق

### 4. ربط شامل بين الإدارات (UI)
- في صفحة تفاصيل العميل: عرض الحجوزات + الفواتير + المدفوعات + الرصيد المتبقي
- في صفحة تفاصيل المورد: عرض الحجوزات الموردة + السدادات + الرصيد
- في صفحة تفاصيل الحجز: تبويب "محاسبي" يعرض القيد + الفاتورة + سداد المورد

### 5. زر "تسوية الحجوزات المستوردة" في صفحة جودة البيانات
- زر واحد يشغّل نفس المنطق على أي حجوزات جديدة مستقبلاً (idempotent — لا يُكرر الفواتير)

## تفاصيل تقنية
- استخدام `INSERT ... WHERE NOT EXISTS` لضمان عدم التكرار
- التريجرات الحالية `trg_invoice_payment_to_journal` و `trg_supplier_payment_journal` تتكفل بالقيود
- صفحة `/data-quality` موجودة بالفعل، نضيف لها قسم "حجوزات بدون أسعار" + زر التسوية
- مكونات جديدة: `CustomerLedgerTab`, `SupplierLedgerTab`, `BookingAccountingTab`

## ما لن يتم
- لن أخمّن أسعار حجوزات ناقصة المبالغ — تحتاج إدخال يدوي
- لن أعدّل القيود التاريخية الموجودة بالفعل، فقط الجديد
