## الخطة

### التشخيص
سبب الخطأ الحالي ليس في رقم الفاتورة ولا العملة، بل في صلاحيات قاعدة البيانات:
- سجل المتصفح يوضح: `new row violates row-level security policy for table "invoices"`
- سياسة الأمان على جدول الفواتير تشترط وجود `organization_id` صالح عند الإدخال
- مكوّن إصدار فاتورة الفندق الحالي يرسل الفاتورة بدون `organization_id`، لذلك يتم رفض الحفظ
- نفس النمط موجود أيضاً في بعض شاشات إنشاء الفواتير الأخرى، لذلك الإصلاح الأفضل يكون شاملًا لنقاط الإنشاء كلها حتى لا يتكرر الخطأ لاحقاً

### ما سأقوم بتنفيذه بعد موافقتك
1. إصلاح إصدار فاتورة الفندق في `HotelInvoiceCreator`
   - إضافة `organization_id` إلى بيانات الإدخال
   - استخدام `organization_id` الخاص بالحجز إن وُجد، مع fallback إلى المؤسسة الحالية من `useOrgId`
   - منع الإرسال إذا تعذر تحديد المؤسسة، مع رسالة خطأ واضحة بدل الرسالة العامة الحالية

2. توحيد نفس الإصلاح في باقي مسارات إنشاء الفواتير
   - `src/components/flight-bookings/FlightInvoiceCreator.tsx`
   - `src/components/invoices/CreateInvoiceDialog.tsx`
   - بحيث كل عملية إنشاء فاتورة تمرر `organization_id` بشكل صريح ومتوافق مع سياسات العزل بين المؤسسات

3. تقوية الأنواع TypeScript للحجوزات
   - إضافة `organization_id` كحقل اختياري في أنواع الحجز المستخدمة في الواجهة
   - حتى يتم الاعتماد على مؤسسة الحجز نفسه بشكل آمن وواضح داخل الكود

4. تحسين رسائل الخطأ
   - إذا كانت المشكلة بسبب عدم وجود مؤسسة حالية أو تعارض صلاحيات، تظهر رسالة مفهومة للمستخدم بدل `خطأ في إصدار الفاتورة` فقط

5. التحقق بعد الإصلاح
   - اختبار إصدار فاتورة من صفحة الحجوزات الفندقية
   - التأكد أن الفاتورة تُنشأ بنجاح
   - التأكد أن `invoice_sent` و `invoice_sent_date` في الحجز يتم تحديثهما بعد نجاح إنشاء الفاتورة

## تفاصيل تقنية
- سياسة الأمان الحالية على `invoices` تشترط أن يكون:
  - `organization_id = ANY(get_user_org_ids(auth.uid()))`
  - ومعها شرط كتابة إضافي عبر `can_org_write(organization_id)`
- لذلك أي `INSERT` بدون `organization_id` أو بقيمة `NULL` سيفشل مباشرة
- يوجد مسار صحيح بالفعل في `src/pages/NewInvoice.tsx` يمرر `organization_id`، وسأجعل باقي المسارات تتبع نفس النمط
- لا يبدو أن الإصلاح الحالي يحتاج تعديلًا في هيكل قاعدة البيانات؛ المشكلة في بيانات الإدخال من الواجهة، لا في schema

```text
Hotel Booking -> Invoice Creator -> add organization_id -> INSERT invoices -> success
```

إذا وافقت، أنفذ الإصلاح مباشرة.
